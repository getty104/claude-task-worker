# Security Policy

## Supported Versions

Only the latest released version of `claude-task-worker` (the most recent version published on [npm](https://www.npmjs.com/package/claude-task-worker)) is supported with security updates. Please make sure you are running the latest version before reporting an issue.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| Older   | :x:                |

## Reporting a Vulnerability

If you believe you have found a security vulnerability in this repository, please report it privately using GitHub's [Private Vulnerability Reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability) feature:

1. Go to the [Security tab](https://github.com/getty104/claude-task-worker/security) of this repository.
2. Click **"Report a vulnerability"** to open a new draft Security Advisory.
3. Fill in as much detail as you can, including steps to reproduce, affected versions, and potential impact.

Please **do not** report security vulnerabilities through public GitHub issues, discussions, or pull requests.

## What to Expect

- We will acknowledge receipt of your report as soon as possible.
- We will investigate and keep you updated on the progress toward a fix.
- Once a fix is available, we will coordinate disclosure with you and credit you (unless you prefer to remain anonymous).

## Scope

This policy covers the `claude-task-worker` CLI (this repository) and the Claude Code plugin bundled under `plugin/`. Vulnerabilities in third-party dependencies should generally be reported upstream, but feel free to let us know if they affect this project so we can update accordingly.
